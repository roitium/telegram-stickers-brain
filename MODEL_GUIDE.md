# Telegram Stickers - 模型使用详细指南

## 🎭 核心概念

这个插件让你能够像真人一样使用 Telegram 表情包，而不是机械地重复发送同一个表情。关键在于：

1. **情感匹配**：理解用户消息的情感，选择合适的情感符号
2. **负载均衡**：同一个情感有多种表达方式，要均匀使用
3. **时机判断**：知道什么时候用表情包比用文字更好

## 🤔 何时调用 `reply_with_sticker`

### ✅ 应该使用的情况

1. **用户明确要求**
   - "发个表情包看看"
   - "来个😂"
   - "sticker plz"
   - "用表情包回复我"

2. **情感强烈回应**
   - 用户说"笑死我了" → 😂
   - 用户说"好难过" → 😭
   - 用户说"什么意思？" → 🤔
   - 用户说"气死了" → 😡

3. **互动游戏**
   - 用户先发表情包 → 用相关表情包回应
   - "sticker battle" 模式

4. **氛围调节**
   - 对话需要轻松一下
   - 打破尴尬气氛
   - 增加趣味性

5. **简短确认**
   - "好的" → 👍
   - "收到" → ✅
   - "明白" → 👌

### ❌ 不应该使用的情况

1. **严肃话题**
   - 讨论重要事情
   - 解决问题时
   - 技术讨论细节

2. **每个回复都用**
   - 保持文字和表情包的平衡
   - 表情包应该是调味品，不是主食

3. **用户情绪低落时**
   - 除非是安慰的表情包（如 😢 🤗）

## 🔄 负载均衡策略详解

### 为什么需要负载均衡？
- 同一个 emoji（如 😂）可能有 10 个不同的表情包
- 如果总是用第一个，会很单调
- 用户会觉得你在重复发送同一个表情

### 系统如何工作？
1. **记录使用次数**：每个表情包被使用的次数
2. **记录最后使用时间**：每个表情包最后一次使用的时间
3. **排序算法**：
   - 优先选择使用次数最少的
   - 如果次数相同，选择更久未使用的
   - 从最合适的几个中随机选择（增加多样性）

### 实际效果
- 第一次用 😂 → 选择使用次数为 0 的表情包 A
- 第二次用 😂 → 选择使用次数为 0 的表情包 B（如果 A 已用）
- 第三次用 😂 → 选择使用次数最少的（可能是 C，或者 A/B 如果其他都用过了）

## 💡 高级使用技巧

### 1. 关键词匹配
除了 emoji，还可以用关键词：
```javascript
reply_with_sticker({ keyword: "laugh" })  // 匹配所有笑相关的
reply_with_sticker({ keyword: "confused" }) // 匹配困惑相关的
```

### 2. 组合使用
```javascript
// 先文字，后表情包
"明白了！" + reply_with_sticker({ emoji: "👌" })

// 先表情包，后文字
reply_with_sticker({ emoji: "🤔" }) + "让我想想..."
```

### 3. 回应特定表情包
如果用户发了 😂，你可以：
- 回同样的 😂（不同款式）
- 回相关的 😄 或 🥰
- 回互动的 😏 或 🤣

### 4. 避免过度使用
- 每 3-5 条消息用一次表情包
- 重要信息不用表情包打断
- 长对话中穿插使用，不要连续使用

## 📝 练习建议

1. **观察真人对话**：注意朋友之间如何使用表情包
2. **情感分析练习**：看到一句话，想想用什么 emoji 回应
3. **多样性练习**：同一个 😂，想想有哪些不同的表达方式
4. **时机判断**：什么时候该用，什么时候不该用

记住：表情包是对话的调味品，让交流更有趣、更生动，但不要让它成为对话的主角。😊
